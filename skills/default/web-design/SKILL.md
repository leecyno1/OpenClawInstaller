# Web Design 审查 Skill

## 目标
对网页或前端界面进行结构化审查，重点覆盖：
1. 信息架构与布局层级
2. 视觉一致性（颜色、字体、间距、组件状态）
3. 交互反馈（空态、加载态、错误态）
4. 可访问性（对比度、键盘可达、语义化）
5. 移动端适配与响应式行为

## 输出格式
- 问题清单（按严重级别排序）
- 每个问题给出具体改法（含 CSS/组件级建议）
- 高优先级修复顺序（P0/P1/P2）

## 执行原则
- 先定位可复现问题，再给建议。
- 每条建议必须可落地，避免抽象描述。
- 与现有设计系统冲突时，优先遵循项目既有规范。
